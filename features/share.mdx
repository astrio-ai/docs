## Share projects with your team

Astrio lets you invite other people to a project and give each person a role. That controls what they can change, whether they can publish, and who can see the project.

## Private and public projects

Every project is either private or public:

- Private: only invited team members with access can open the project.
- Public: the project can be shared or discovered outside that closed group (for example via a link or public listing), depending on how you configure it in Astrio.

Projects are public by default and can be set to private by Pro or above plan users.

## What each role can do

This table shows which actions each role can take on a project.

| Capability | Owner | Editor | Viewer |
| :--- | :--- | :--- | :--- |
| Preview | Yes | Yes | Yes |
| Edit project | Yes | Yes | No |
| Deactivate / delete project | Yes | No | No |
| Invite others | Yes | Yes | No |
| Transfer ownership | Yes | No | No |
| Payments | Yes | Yes | No |
| Publish | Yes | Yes | No |

<Warning>
  Review who has owner or editor access before sharing sensitive content. Viewers cannot publish or edit, but editors can change the project and the live site when they publish.
</Warning>
