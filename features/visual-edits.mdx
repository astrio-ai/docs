## Visual editor overview

In the Astrio editor, the preview shows your site as visitors will see it. Click an element on the page to select it (it is highlighted on the canvas). An Edit panel opens so you can change that element directly.

On the left, the chat sidebar is where you talk to Astrio about bigger or multi-step changes. Visual edits focus on one selected element at a time in the Edit panel; chat is there when you prefer to describe changes in natural language.

## How it works

<Steps>
  <Step title="Select an element">
    Click something in the preview—headline, button, image, section, or other UI. The selection is outlined so you know what you are editing.
  </Step>
  <Step title="Use the Edit panel">
    The Edit panel shows controls for the selection. Edit text in place when the element supports it. Use the Style and Layout tabs to adjust appearance and structure. Open sections such as Colors, Typography, Border, and Display to fine-tune values (for example text and background colors with hex codes or swatches).
  </Step>
  <Step title="Preview updates">
    Changes apply in the preview as you adjust settings so you can see the result immediately.
  </Step>
  <Step title="Keep iterating">
    Select another element to keep editing, or use chat for requests that span multiple parts of the page or need AI help.
  </Step>
</Steps>

## Chat vs the Edit panel

- Edit panel: Best for direct tweaks to the currently selected element—colors, type, borders, display options, layout fields, and text where available.
- Chat: Best when you want to describe a change in words, work across the page, or iterate with Astrio in conversation.

Astrio does not surface a separate “smart suggestions” list inside the editor. You drive changes by selecting elements, adjusting controls, and typing in chat when that fits your workflow.

## Tips

- Select the smallest element that matches what you want to change (for example the headline, not the whole hero) so the Edit panel shows the right fields.
- If a section is collapsed in the panel, expand it to reveal typography, border, or display options.
- After large layout or style changes, check the preview at different breakpoints if your project supports responsive preview.

<Warning>
  Big structural changes can affect how the page behaves on small screens. Re-check the preview after significant layout edits.
</Warning>
